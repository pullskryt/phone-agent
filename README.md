# Termux AI Agent

CLI-агент вроде Claude Code, работающий прямо в Termux. Поддерживает:
- **API-режим**: Groq / OpenAI / любой OpenAI-совместимый эндпоинт
- **Local-режим**: локальная gguf-модель через llama.cpp сервер (бесплатно, но медленнее и слабее)

## Telegram-бот

Тот же агент, но через Telegram — пишешь боту из любого места, не нужно
держать открытым Termux с терминалом на виду.

Настраивается в `install.sh` (спросит токен от @BotFather и твой Telegram
user_id, который можно узнать у @userinfobot), либо вручную в `config.json`
→ `telegram`. Бот отвечает **только** тем ID, что в `allowed_user_ids` —
пока список пуст, никому не отвечает.

Запуск:
```bash
aibot
```

Работает изнутри Termux (long polling), значит жив ровно пока жив процесс —
как и обычный CLI-агент. Закрыл Termux — бот остановился.

Подтверждения (`run_python`, `delete_file`, `install_package`, опасные
bash-команды) в Telegram приходят как сообщения с кнопками "Разрешить" /
"Отклонить" — работает так же строго, как y/n в терминале.

## Установка

```bash
cd termux-agent
chmod +x install.sh
./install.sh
```

Скрипт сам спросит: какой облачный сервис (Groq / свой OpenAI-совместимый URL / пропустить),
API-ключ, модель, и предложит сразу поставить локальную модель через Ollama — всё
интерактивно, без ручного редактирования `config.json`. Заодно создаёт `~/Projects`
(см. ниже).

## Папка Projects

Когда просишь агента создать новый проект (бота, скрипт, приложение) без явного
указания пути, он сам кладёт его в `~/Projects/<название>/` — так все созданные
им вещи собраны в одном месте, а не разбросаны по домашней директории. Если
называешь путь явно ("сделай в ~/scripts/foo.py") — использует именно его.

После установки:

```bash
source ~/.bashrc
```

и дальше из любой папки Termux просто:

```bash
ai
# или
ии
```

Оба запускают агента с рабочей директорией = той папке, где ты сейчас находишься
(агент читает/пишет файлы именно там, а не в папке самого проекта).

Если нужно настроить алиасы отдельно (без полной установки):
```bash
bash setup_alias.sh
```

## Запуск напрямую (без алиаса)

```bash
python agent.py --provider api
```

## Локальный режим (бесплатно, офлайн) — теперь одна команда

С 2026 года Ollama ставится в Termux напрямую через `pkg`, без ручной сборки
llama.cpp. Всё делает один скрипт:

```bash
bash local_setup.sh
```

Он сам: поставит Ollama, запустит сервер в фоне, спросит какую модель скачать
(по умолчанию `qwen2.5:3b` — баланс скорости/качества для среднего телефона),
скачает её и пропишет в `config.json`.

После этого:

```bash
ai --provider local
```

Если перезапустил Termux и сервер Ollama погас — подними его заново одной командой:

```bash
bash local_start.sh
```

(`local_setup.sh` нужен только один раз при первой установке.)

## Честно о локальном режиме

Маленькие модели (1.5B-4B), которые реально влезут в телефон без зависаний,
**хуже справляются с tool calling**, чем большие облачные модели — они чаще:
- вызывают инструменты с кривыми аргументами
- путаются после нескольких шагов подряд

Агент теперь сам умеет восстанавливаться после части таких ошибок (просит
модель повторить попытку), но полностью проблему это не убирает — маленькая
модель физически слабее. Local-режим — хороший бесплатный офлайн-вариант для
простых задач, но для сложной многошаговой работы `--provider api` с Groq
пока стабильнее.

## Инструменты агента (для ИИ)

`read_file` · `write_file` · `edit_file` · `list_dir` · `delete_file` ·
`bash_exec` · `run_python` · `check_syntax` · `grep_search` · `web_fetch` ·
`install_package` · `list_dialogues` · `read_dialogue`

`run_python`, `delete_file` и `install_package` **всегда** требуют твоего
подтверждения (y/n) — модель не может их выполнить незаметно, в отличие от
обычного bash_exec, где подтверждение спрашивается только для явно
разрушительных команд.

## Память о прошлых диалогах

При выходе (`exit` или `/clear`) агент сохраняет диалог в `~/Dialogues` как
читаемый текстовый файл — хранятся только последние 5, старые удаляются
автоматически. Модель сама решает, когда ей нужен прошлый контекст: если
текущий запрос ссылается на что-то из прошлой сессии, она вызывает
`list_dialogues`, выбирает подходящий по дате в имени файла и читает через
`read_dialogue`. Если контекст не пригодился — просто не использует его дальше.

Учти: раз это решает модель сама, а не только по явному запросу, в редких
случаях она может прочитать прошлый диалог, когда это не требовалось — это
тратит немного лишних токенов, но не должно влиять на итоговый ответ.

## Команды (для тебя)

- `/help` — список всех инструментов и команд
- `/run <file> [args]` — запустить python-файл напрямую, без обращения к модели
- `/model <name>` — сменить модель на лету (например `/model qwen/qwen3.6-27b`)
- `/system [текст]` — показать текущий системный промпт или заменить его
- `/provider` — показать текущий провайдер
- `/logs` — вкл/выкл подробный вывод вызовов инструментов (тише/подробнее)
- `/clear` — очистить историю диалога и экран, не выходя из агента

## Структура

```
agent.py              — главный цикл (диалог + tool calling)
tools.py               — инструменты для ИИ
display.py             — цвета/рамки/спиннер/маскот в терминале
telegram_bot.py         — Telegram-интеграция (переиспользует agent.py)
providers/api.py       — облачный провайдер (Groq и т.п.)
providers/local.py     — локальный провайдер (Ollama)
config.json             — настройки: провайдер, ключи, модели, системный промпт
setup_alias.sh          — настройка команд ai/ии
local_setup.sh          — установка Ollama + модели одной командой
local_start.sh          — быстрый повторный запуск Ollama сервера
```

## Безопасность

Опасные bash-команды (`rm -rf`, `mkfs`, `dd if=` и т.п.) требуют
подтверждения `y` перед выполнением. Остальное выполняется без спроса —
агент работает в твоей рабочей директории с твоими правами, будь аккуратен.
