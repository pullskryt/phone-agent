# Termux AI Agent

CLI-агент вроде Claude Code, работающий прямо в Termux. Поддерживает:
- **API-режим**: Groq / OpenAI / любой OpenAI-совместимый эндпоинт
- **Local-режим**: локальная gguf-модель через llama.cpp сервер (бесплатно, но медленнее и слабее)

## Установка

```bash
cd termux-agent
chmod +x install.sh
./install.sh
```

Впиши свой API-ключ в `config.json` → `api.api_key`.
Groq — хороший выбор для API, бесплатный тир есть, ключ берётся на console.groq.com.

`install.sh` сам настраивает быстрый запуск — после установки выполни:

```bash
source ~/.bashrc
```

и дальше из любой папки Termux просто:

```bash
ai
# или
ии
```

Оба запускают агента с рабочей директорией = той папке, где ты сейчас находишься
(агент читает/пишет файлы именно там, а не в папке самого проекта).

Если нужно настроить алиасы отдельно (без полной установки):
```bash
bash setup_alias.sh
```

## Запуск напрямую (без алиаса)

```bash
python agent.py --provider api
```

## Локальный режим (бесплатно, без интернета)

### 1. Собери llama.cpp

```bash
pkg install cmake git clang -y
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
cmake -B build
cmake --build build --config Release -j4
```

Сборка на телефоне может занять 10-20 минут в зависимости от процессора.

### 2. Скачай маленькую gguf-модель

Для телефона реально тянут только маленькие модели (1.5B–4B, квантованные Q4).
Хорошие варианты с поддержкой tool calling:
- **Qwen2.5-3B-Instruct-GGUF** (Q4_K_M) — неплохо понимает function calling
- **Qwen2.5-1.5B-Instruct-GGUF** — быстрее, но глупее

```bash
mkdir -p ~/models
cd ~/models
# скачай .gguf файл через termux-download или wget с huggingface
wget https://huggingface.co/<repo>/resolve/main/model.gguf
```

### 3. Запусти сервер

```bash
cd llama.cpp
./build/bin/llama-server -m ~/models/model.gguf -c 4096 --port 8080
```

Оставь это в отдельной Termux-сессии (или через `tmux`/`screen`), а в другой запускай агента:

```bash
python agent.py --provider local
```

## ⚠️ Честно о локальном режиме

Маленькие модели (1.5B-4B), которые реально влезут в телефон без зависаний,
**плохо справляются с tool calling** — они часто:
- вызывают инструменты с кривыми аргументами
- забывают формат JSON
- путаются после 3-4 шагов

Для реальной работы (как у Claude Code) практического качества с локалки на
телефоне добиться сложно — железо телефона просто не тянет модели того класса,
которые нормально держат агентные сценарии. Так что local-режим — это скорее
"поиграться и есть бесплатный фоллбек", а не полноценная замена API.

Если нужен рабочий инструмент — используй `--provider api` с Groq
(там есть быстрые и бесплатные модели вроде llama-3.3-70b с щедрым лимитом).

## Инструменты агента (для ИИ)

📖 `read_file` · ✏️ `write_file` · ✂️ `edit_file` · 📂 `list_dir` · 🗑️ `delete_file`
💻 `bash_exec` · 🐍 `run_python` · 🔍 `check_syntax` · 🔎 `grep_search` · 🌐 `web_fetch`

`run_python` и `delete_file` **всегда** требуют твоего подтверждения (y/n) —
модель не может их выполнить незаметно, в отличие от обычного bash_exec,
где подтверждение спрашивается только для явно разрушительных команд.

## Команды (для тебя)

- `/help` — список всех инструментов и команд
- `/run <file> [args]` — запустить python-файл напрямую, без обращения к модели
- `/model <name>` — сменить модель на лету (например `/model qwen/qwen3.6-27b`)
- `/provider` — показать текущий провайдер
- `/clear` — очистить историю диалога (контекст), не выходя из агента

## Структура

```
agent.py              — главный цикл (диалог + tool calling)
tools.py               — инструменты: read_file, write_file, edit_file, list_dir, bash_exec
providers/api.py       — облачный провайдер
providers/local.py     — локальный провайдер (llama.cpp сервер)
config.json             — настройки: провайдер, ключи, модели, системный промпт
```

## Безопасность

Опасные bash-команды (`rm -rf`, `mkfs`, `dd if=` и т.п.) требуют
подтверждения `y` перед выполнением. Остальное выполняется без спроса —
агент работает в твоей рабочей директории с твоими правами, будь аккуратен.
